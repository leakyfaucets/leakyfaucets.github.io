本文通过利用`Powershell`别名来使`VSCode`能够正常激活`mamba`管理的虚拟环境。

## 问题描述

`VSCode`的`Python`插件默认使用`conda activate <env_name>`来激活虚拟环境。由于`mamba`创建的环境使用`conda`命令无法激活，这导致`VSCode`在切换或激活环境时直接报错，环境始终无法成功加载。

报错示例：
```
EnvironmentNameNotFound: Could not find conda environment: py
You can list all discoverable environments with `conda info --envs`.
```

## 解决方案：利用`Powershell`别名

### 操作步骤

1. 定位`PowerShell`配置文件：`VSCode`在`Windows`上默认使用`PowerShell`。我们需要修改其启动配置文件 `profile.ps1`，让配置在每次打开终端时自动生效。

   配置文件路径通常为：
   ```
   $HOME/Documents/WindowsPowerShell/profile.ps1
   ```
   (注：`$HOME`为 `C:\Users\你的用户名`)，或使用
   ```powershell
   code $Profile
   ```
   在`VSCode`中打开。

2. 添加别名代码：在打开的 `profile.ps1` 文件末尾，添加以下一行代码：
   ```powershell
   # 将 conda 命令映射到 mamba
   Set-Alias -Name conda -Value mamba
   ```

3. 保存`profile.ps1`文件

4. 修改Python扩展设置：在`VSCode`中设置`python.condaPath: mamba`。

5. 重启终端

6. 验证修复：通过`VSCode`右下角的解释器切换功能激活环境，此时应该能顺利执行`conda activate`。
